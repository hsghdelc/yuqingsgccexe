# 国网人资舆情筛查系统 - 产品需求文档 (PRD)

**版本：** v1.3.0
**日期：** 2026年7月4日
**状态：** 已完成开发

---

## 一、产品概述

### 1.1 产品定位

国网人资舆情筛查系统是一款面向国家电网公司人力资源部门的专业舆情监测工具，用于自动化筛查招聘和薪酬管理相关的负面舆情信息。

### 1.2 目标用户

- 国家电网公司人力资源部门工作人员
- 各省/市电力公司人事管理人员
- 舆情监测分析人员

### 1.3 核心价值

- **效率提升**：将人工逐条审核转化为自动化批量筛查，处理速度提升数十倍
- **精准识别**：通过"实体词+领域词+情感词"三层匹配机制，降低误报率
- **AI 辅助**：集成大语言模型进行二次核验，进一步提升准确率
- **可视化导出**：三色关键词高亮标注，直观展示命中信息
- **词库逆向**：通过 AI 智能体自动从舆情数据中提取高价值词根，持续优化识别规则

## 二、技术架构

### 2.1 技术栈

| 层级 | 技术 |
|---|---|
| 前端框架 | 纯 HTML/CSS/JavaScript（单文件架构） |
| Excel 读取 | SheetJS (xlsx.full.min.js) |
| Excel 写入 | ExcelJS (exceljs.min.js) - 支持富文本和样式 |
| AI 接口 | OpenAI 兼容 API（DeepSeek/自定义模型） |
| 智能体 API | Dify Workflow API（词库逆向分析） |
| 网页抓取 | Node.js http/https + cheerio + Electron BrowserWindow |
| 桌面打包 | Electron + electron-builder (NSIS/Portable) |
| 部署平台 | Vercel（网页版）/ Windows（桌面版）

### 2.2 部署模式

| 模式 | 说明 |
|---|---|
| **网页版** | 部署在 Vercel，支持基础识别功能，URL 内容解析和词库逆向不可用 |
| **桌面版 (exe)** | Electron 打包，支持全部功能，含 URL 内容解析、词库逆向和登录持久化 |

## 三、核心功能模块

### 3.1 系统首页

**功能描述：** 展示系统欢迎信息和标准操作流程

**设计亮点：**
- 内嵌国网 SVG Logo（离线可用，无外部依赖）
- 四步操作流程可视化卡片（导入→配置→扫描→导出）
- 深色极客风格 UI，与系统整体一致

### 3.2 识别配置

#### 3.2.1 本地模式

**核心规则：** 实体词 AND 领域词 AND 负面情感词 三者同时命中

**设计亮点 - 实体词分层机制：**
- **强实体**：含"国网/国家电网"、具名子公司（南瑞/许继等）、≥5字的机构名 → 出现即确认主体
- **弱实体**：供电局、省电力等泛化词 → 须与领域词邻近共现
- **邻近共现距离**：可调参数（默认30字），弱实体与领域词间隔超过此距离不算命中
- **强实体豁免词**：用户可手动指定短词为强实体（如"冀电"）

**阈值调节：** 滑块式阈值（0.01-0.50），支持实时预览

#### 3.2.2 AI 模式

**工作流程：**
1. 先运行本地模式完成全量扫描
2. 将本地标记为"是"的行发送给 LLM 二次核验
3. 并发执行，支持多 Key 轮询

**LLM Prompt 设计：**
- 精简版分类体系（招聘类/薪酬类）
- 支持 JSON 格式输出
- 1500 字安全截断，防止超出模型上下文

**支持的模型：**
- DeepSeek Chat（内置）
- 自定义模型（用户可添加）

### 3.3 词库管理

**词库分类：**
| 词库 | 说明 | 颜色标识 |
|---|---|---|
| 实体词库 | 国网相关主体词 | 蓝色 |
| 领域词库 | 招聘·薪酬·劳动关系词 | 紫色 |
| 负面情感词库 | 负面情感·行为·评价词 | 红色 |

**设计亮点 - 实体词强弱高亮：**
- 使用"彩色背板 + 透明文字 textarea"叠加方案
- 强实体显示绿色，弱实体显示黄色
- 用户新增词时系统自动按规则判定强弱并实时染色
- 无需用户手动指定，规则统一

**词库持久化：**
- 使用 localStorage 保存
- 词库版本号检测，更新词库时自动重置为新默认值
- 用户修改后点击"确认词库修改"才会保存

### 3.4 识别结果

**结果展示：**
- 预览表格显示：行号、标题、摘要、命中词（三类分色）、AI 确认
- 命中词分三类显示：实体词（蓝色）、领域词（紫色）、情感词（红色）
- 颜色与词库管理页的卡片标题色一致

**导出功能：**
- 使用 ExcelJS 保留原文件格式
- 关键词级富文本着色（单元格内分段着色）
- B 列"是"：本地=黑色，AI确认=红色加粗
- 跳过公式单元格，避免破坏底层 XML

### 3.5 运行日志

- 实时显示系统运行状态
- 支持清空日志

### 3.6 字段映射

**功能：** 用户可自定义分析列，覆盖系统默认的"标题/摘要"识别规则

**设计：**
- 默认 6 个下拉框（3行2列 Grid 布局）
- 上传文件后自动填充所有标题
- 不选择则使用系统默认规则
- 选择后系统优先分析用户指定的列

### 3.7 内容解析（桌面版专属）

**功能：** 从 Excel 中的 URL 链接自动提取网页文字内容

**三层递进抓取架构：**

| 层级 | 方案 | 速度 | 覆盖场景 |
|---|---|---|---|
| ① Node.js http + cheerio | 快速提取 | 1-2秒/条 | 新闻门户、政府网站、微信公众号 |
| ② 隐藏 BrowserWindow | SPA 渲染 | 5-8秒/条 | 微博正文、头条文章、小红书 |
| ③ 弹窗 BrowserWindow | 用户登录 | 一次性 | 需登录才能看的站点 |

**前置登录功能：**
- 支持微博、今日头条、抖音、知乎、百家号、小红书
- 用户可提前登录，Cookie 自动持久化
- 解析过程中遇到需登录的网站自动弹窗

**并发控制：**
- http 请求：默认 20 并发（可调 5-50）
- BrowserWindow：单窗口复用（内存控制）

**写入规则：**
- 固定新增列"抓取原文"（maxCol + 1）
- 所有结果垂直对齐，避免锯齿状数据
- 提取失败标记为"提取失败"

**网页版提示：**
- 显示"网页端暂未开放，请通过桌面端使用"
- 提供下载桌面端按钮（链接到 GitHub）

### 3.8 词库逆向（桌面版专属）

**功能：** 通过 Dify AI 智能体，从舆情数据中自动逆向提取高价值词根

**技术架构：**

```
用户上传 Excel → 本地 XLSX 解析 → 转为纯文本 → 发送给 Dify API → 获取结果 → 展示表格
```

**工作流程：**

1. **文件上传**：用户上传包含舆情文本和正负面标签的 Excel 文件
2. **本地解析**：使用 SheetJS 解析 Excel，提取为制表符分隔的纯文本
3. **API 调用**：将纯文本发送给 Dify 工作流 API
4. **结果解析**：解析 Dify 返回的 Markdown 表格
5. **展示结果**：在表格中展示提取的词根及分析数据

**Dify 工作流：**

| 步骤 | 节点 | 说明 |
|---|---|---|
| 1 | 开始 | 接收纯文本输入（变量名：query） |
| 2 | 代码执行1：分批器 | 将长文本切分为适合 LLM 处理的小块 |
| 3 | 迭代节点（LLM） | 并发调用大模型进行高敏词汇挖掘 |
| 4 | 代码执行2：全局聚合 | 汇聚词汇进行交叉校验、频次统计、风险评分 |
| 5 | 结束 | 输出 Markdown 格式的词根报表 |

**输出格式：**

| 关键词 | 负面频次 | 非负面频次 | 负面浓度 | 风险得分 | 修正危机指数 | 词汇分类 |
|:---|:---:|:---:|:---:|:---:|:---:|:---|
| 贪污腐败 | 1 | 0 | 100.0% | 95 | 95 | 负面情感词 |
| 克扣 | 1 | 0 | 100.0% | 90 | 90 | 负面情感词 |
| 内退 | 1 | 2 | 33.3% | 70 | 23 | 领域词 |

**词汇分类：** 实体词、领域词、负面情感词

**模板文件：**

| content | label |
|---|---|
| （用户填写舆情文本） | 1 或 0（负面填1，正面填0） |

**导出功能：**
- 点击右上角"导出结果"按钮
- 导出为"逆向分析结果.xlsx"
- 包含完整的 7 列分析数据

**网页版提示：**
- 显示"网页端暂未开放，请通过桌面端使用"
- 提供下载桌面端按钮（链接到 GitHub）

## 五、核心算法设计巧思

### 5.1 实体词分层机制（设计亮点）

**问题：** 传统实体词匹配过于宽泛，"电力""电网"等泛词会导致大量误报

**解决方案：**
- 强实体（含"国网"、具名子公司、≥5字）→ 出现即确认主体
- 弱实体（供电局、省电力等泛词）→ 须与领域词邻近共现
- 邻近共现距离可调（默认30字）

**效果：**
- "电力行业改革...（隔500字）...某机构招聘黑幕" → 放过（误报消除）
- "国家电网招聘内定黑幕" → 命中（真负面不漏）
- "供电局招聘克扣待遇" → 命中（弱实体邻近领域词）

### 5.2 强实体豁免机制（设计亮点）

**问题：** 用户添加的短简称（如"冀电"）不含"国网"且不足5字，会被判为弱实体

**解决方案：**
- 提供"强实体豁免词"输入框
- 填入的词强制判为强实体，绕过常规规则
- 实体词库实时高亮显示强弱分类（绿色/黄色）

### 5.3 关键词级富文本着色（设计亮点）

**问题：** SheetJS 社区版不支持单元格样式和富文本

**解决方案：**
- 改用 ExcelJS 实现
- `buildRichText` 函数将单元格文本按命中关键词切成多段
- 仅关键词段染红，其余文字保持默认色
- 三类词颜色：实体蓝 `#60a5fa`、领域紫 `#a78bfa`、情感红 `#ef4444`

### 5.4 时间分片防假死（性能优化）

**问题：** ExcelJS 导出时主线程阻塞，UI 卡死

**解决方案：**
- 每渲染 50 行，通过 `await sleep(0)` 主动释放主线程
- 进度条实时更新（30%→70%）
- 用户感知不到卡顿

### 5.5 词库版本检测（设计亮点）

**问题：** 用户保存的旧词库会覆盖新默认词库

**解决方案：**
- 引入 `LEXICON_VERSION` 版本号
- 保存配置时记录版本号
- 加载时检测版本：不匹配则使用新默认词库
- 用户自定义词库正常保存/恢复

## 六、数据流架构

### 6.1 识别流程

```
用户上传 .xlsx
    ↓
SheetJS 读取 → 提取表头 → 填充字段映射下拉框
    ↓
用户点击"开始识别"
    ↓
本地扫描（实体词+领域词+情感词 AND 匹配）
    ↓
[AI模式] → 并发发送给 LLM 二次核验
    ↓
ExcelJS 写入标注文件（三色关键词高亮）
    ↓
用户导出 .xlsx
```

### 6.2 内容解析流程

```
用户选择 URL 列
    ↓
可选：前置登录常用网站
    ↓
点击"开始解析"
    ↓
http 请求快速提取 → 失败 → BrowserWindow 渲染提取
    ↓                         ↓
成功 → 写入结果         检测到登录页 → 弹窗用户登录
    ↓                                   ↓
                                登录成功 → Cookie 保存 → 重试
    ↓
写入"抓取原文"列 → 导出
```

## 八、安全性设计

### 8.1 API Key 保护

- 纯前端 BYOK 模式（Bring Your Own Key）
- Key 仅存在用户浏览器 localStorage，不上传服务器
- Vercel 部署时 Key 不会泄露到服务端

### 8.2 登录凭据保护

- 密码使用哈希比较（非明文存储）
- localStorage 存储 base64 编码（非明文，但可逆）
- Cookie 持久化到 Electron userData 目录

### 8.3 网页端安全

- 文件上传功能不依赖外部服务
- 所有处理在浏览器端完成
- 无后端数据库，无用户数据收集

## 附录：开发过程中的关键设计决策

| 决策 | 理由 |
|---|---|
| SheetJS → ExcelJS | SheetJS 社区版不支持单元格样式和富文本 |
| 实体词分层（强/弱） | 泛词（电力/电网/电工）导致大量误报 |
| 邻近共现机制 | 防止"长文中两个词碰巧都出现"的误判 |
| 强实体豁免词 | 用户自定义短简称的强实体判定 |
| http + BrowserWindow 双层抓取 | SSR 页面快速提取 + SPA 页面渲染提取 |
| 前置登录 + Cookie 持久化 | 解决需登录网站的内容提取问题 |
| 时间分片导出 | 防止大文件导出时 UI 卡死 |
| 词库版本检测 | 防止用户旧词库覆盖新默认词库 |
| localStorage + try-catch | 兼容 file:// 协议的安全限制 |
